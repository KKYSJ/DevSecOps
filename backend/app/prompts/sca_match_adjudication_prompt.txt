Use these decision meanings exactly:
- same: the two findings describe the same underlying issue for this stage
- related: the findings are security-related and nearby, but not clearly the same vulnerability
- different: the findings describe different vulnerabilities or unrelated issues

Stage-specific guidance:

- The same package with different CVEs, or different packages with the same weakness class, should usually be `different` or at most `related`.
- Version differences can still be `same` when the package and CVE clearly align.
